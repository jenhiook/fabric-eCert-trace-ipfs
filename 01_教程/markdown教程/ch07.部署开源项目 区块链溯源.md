## 一、简介

本教程将指导您如何在CentOS系统上部署基于Hyperledger Fabric V2.5的通用溯源项目。项目来源于开源社区，通过本教程，您将掌握从服务器环境准备到项目部署的整个流程。
## 二、环境准备
### 1. 克隆项目
首先，克隆项目到本地：
```bash
git clone https://gitee.com/real__cool/fabric-trace
cd fabric-trace
```
### 2. 安装依赖
确保您的服务器已安装以下依赖：
- Docker
- Git
- Go语言环境
- Node.js和npm
## 三、启动区块链网络
### 1. 修改start.sh和stop.sh
修改`fabric-trace/blockchain/network`目录下的`start.sh`文件：
```bash
image_versions=("2.5.9")
```
屏蔽以下内容：
```bash
# 启动mysql容器
#docker run --name fabrictrace-mysql -p 3337:3306 -e MYSQL_ROOT_PASSWORD=fabrictrace -d mysql:8
```
修改`fabric-trace/blockchain/network`目录下的`stop.sh`文件：

屏蔽以下内容：

```bash
#docker rm -f fabrictrace-mysql > /dev/null 2>&1
```



### 2. 启动区块链网络

在`fabric-trace/blockchain/network`目录下执行：
```bash
./start.sh
```
## 四、启动后端
### 1. 修改后端mysql端口
在`/root/fabric-trace/application/backend/settings`找到`config.yaml`，修改数据库信息为：
```yaml
mysql:
  host: "127.0.0.1"
  port: 3337
  user: "sugar20240713"
  password: "sugar20240713"
  charset: "utf8mb4"
  db: "sugar20240713"
```
### 2. 修改前端IP
替换以下文件中的IP地址为您的云服务IP：
- `fabric-trace/application/web/.env.production`
- `fabric-trace/application/web/.env.development`
- `fabric-trace/application/web/src/router/index.js`
### 3. 启动后端
在`fabric-trace/application/backend`目录下执行：
```bash
go run main.go
```
## 五、启动前端
### 1. 安装依赖
新开一个SSH窗口，在`fabric-trace/application/web`目录下执行：
```bash
npm install
```
### 2. 启动前端
在`fabric-trace/application/web`目录下执行：
```bash
npm run dev
```
## 六、访问系统
在浏览器中打开：`http://云服务器IP:9528`，即可看到前端页面。
## 七、测试上链
### 1. 测试区块链浏览器
登录`107.173.182.252:8080/#/`（替换为您的IP地址）

通过本教程，您已成功在CentOS系统上部署了基于Hyperledger Fabric V2.5的通用溯源项目。从服务器环境准备到项目部署，每一步都为您详细讲解。接下来，您可以在此基础上进行二次开发，以满足您的业务需求。如有疑问，请随时查阅官方文档或参与社区讨论。
